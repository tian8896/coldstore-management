Warehouse 2

此处为 第二座冷库 的独立管理域预留入口：可与 Warehouse 1 使用不同的流程、规则与数据库路径（例如独立的 Realtime DB 前缀或子项目）。

与 Warehouse 1、公司财务不共享核心表；将来若要对账或汇总，可通过引用字段（采购单号、入库单号、金额凭证 ID 等）或只读报表拉通数据，而不是直接写入对方业务库。

Independent domain from W1 & Finance; optional future links via references / ETL / aggregated views.

公司财务 · Company Financial

此处预留 公司级财务 的独立模块（总账、应收应付、报表、对账等），与两座冷库的仓储业务数据域分离。

将来若财务需要引用冷库侧数据，建议只保存外部引用（例如业务单号、金额快照、对账批次号），需要实时明细时再按需查询或同步，以保持各域边界清晰、审计可追溯。

Independent from W1/W2 ops data; future ties via document IDs, amounts snapshots, or controlled sync—not tight table coupling.
